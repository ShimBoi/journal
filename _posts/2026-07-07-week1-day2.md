Since JAX naturally doesn't support in-place mutation, to change the value of an element in an array requires allocating a new array:
```
arr = jnp.array([1, 2, 3])
arr[0] = 0  # ERROR
arr.at[0].set(0)
arr.at[0].add(-3)
```

Seems a bit verbose to have to use .at() and .set()/.add(), but it makes sense to clearly distinguish that it's not an in place mutation. In practice, since the reference to the old array is unused, adding jax.jit will optimize it to an in-place mutation, but if the old array is used then it might do a full copy?
