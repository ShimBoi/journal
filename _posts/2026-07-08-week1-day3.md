JAX doesn't use global random state so it's explicitly tracked via a `key` using jax.random.key(42). Anything that uses the random function takes in the key to produce a deterministic generation. Reusing keys is not recommended since they produce exactly the same output (unless explicitly required), and to get a new key, use jax.random.split which produces 2 new keys:
```
key1, key2 = jax.random.split(key)
```

If there was a global random key like in numpy, then running the same method with the same key reference would produce different results make the function not pure.
