Read the "Thinking in JAX" tutorial today. Rough notes, not cleaned up:

- Pure function = same input, same output, no side effects. JAX needs this because `jit` traces a function once and reuses the compiled version — if the function had side effects, the cached trace could go stale silently.
- The tell for "impure": if calling it twice with the same input could give a different output (global counter, unpassed RNG, print statements affecting control flow).
- Still not 100% sure how this interacts with Python-level debugging — need to come back to `jax.debug.print` later.
